1. Norman’s Model (The Execution–Evaluation Cycle)

Norman’s model breaks interaction into two primary phases (Execution and Evaluation) across seven distinct stages.

The Seven Stages of Action

  1. Goal: The high-level objective (e.g., "I need more light.")
  2. Intention: The decision to act (e.g., "I will turn on the light switch.")
  3. Specify: Selecting the specific actions (e.g., "I will move to the switch and prepare to press it.")
  4. Execute: The physical performance of the action.
  5. Perceive: Observing the result (e.g., Seeing the light turn on).
  6. Interpret: Processing the observation (e.g., "The room is bright; the light is now on.")
  7. Evaluate: Comparing the result with the original goal (e.g., "Mission complete.")

Purpose: To identify the Gulf of Execution (the gap between goals and system actions) and the Gulf of Evaluation (the gap between system state and user understanding).

Note: Norman's model is purely cognitive and does not account for user expertise level or predict execution time.

2. GOMS Model

A cognitive model used to predict the estimated time for task completion by analyzing routine performance.

  • G (Goals): What the user wants to achieve.
  • O (Operators): Basic mental or physical actions performed.
  • M (Methods): The procedures/steps used to achieve a goal.
  • S (Selection Rules): The logic used to choose a specific method when multiple options exist.

Key Distinction: Unlike Norman, GOMS does not model the Specify, Perceive, Interpret, or Evaluate stages. It assumes the user is an expert performing routine tasks without error.

3. KLM Model (Keystroke-Level Model)

KLM is a low-level, physical/device-based version of GOMS. It is the most granular level of prediction in this hierarchy.

KLM Characteristics

  • Strict Focus: Focuses strictly on Operators (e.g., keystrokes, mouse clicks, pointing, and mental preparation time).
  • Simplification: It removes higher-level goals, methods, and selection rules to provide a raw time estimate.
  • Unit of Measure: Performance is predicted in seconds based on established constants (e.g., [K] = 0.2s, [M] = 1.35s).

Worked Example: Deleting a File

This example compares two methods for deleting a file, assuming the user starts with their hand on the mouse.

Method A: Drag to Trash (Mouse Only)

Step Operator Time (s)
1. Mentally prepareM1.35
2. Point to fileP1.10
3. Click and holdB[down]0.10
4. Drag to Trash iconP1.10
5. Release buttonB[up]0.10
Total3.75s

Method B: Keyboard Shortcut (Ctrl+D)

Step Operator Time (s)
1. Mentally prepareM1.35
2. Point to fileP1.10
3. Click to selectB[click]0.20
4. Move hand to keyboardH[kbd]0.40
5. Press Ctrl+D (2 keys)2K0.40
Total3.45s

Analysis: While Method B introduces a Homing (H) penalty for moving the hand from mouse to keyboard, it remains faster because dragging a mouse across a screen (long-distance Pointing) is physically more time-consuming than rapid keystrokes.
